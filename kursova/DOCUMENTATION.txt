ДОКУМЕНТАЦІЯ ПРОГРАМИ VIRTUAL STREAM DECK
=========================================

1. Загальний опис
-----------------

Virtual Stream Deck - це Windows-програма, яка створює невелику панель швидких кнопок.
Панель можна викликати глобальною комбінацією клавіш Ctrl + Alt + Space.

Кожна кнопка може виконувати одну з дій:
- відкрити сайт;
- запустити програму або файл;
- відкрити папку;
- виконати гарячу клавішу, наприклад ctrl+c;
- вставити текст;
- перейти на іншу сторінку кнопок;
- бути вимкненою.

Програма працює у фоні та відображається в прихованих піктограмах Windows.
Через іконку в треї можна показати панель, відкрити налаштування або повністю вийти з програми.


2. Основні можливості
---------------------

2.1. Панель кнопок

Панель показує кнопки у вигляді сітки.
Розмір сітки можна змінювати:
- 3 x 3;
- 4 x 4;
- 5 x 5;
- 6 x 6.

Панель може розташовуватись у різних місцях екрана:
- зверху зліва;
- зверху по центру;
- зверху справа;
- зліва по центру;
- по центру;
- справа по центру;
- знизу зліва;
- знизу по центру;
- знизу справа.

Панель має анімацію:
- при відкритті вилітає знизу;
- при закритті ховається вниз.

Якщо користувач натискає мишкою поза панеллю або переходить в іншу програму через Alt + Tab,
панель автоматично згортається.


2.2. Гарячі клавіші

Ctrl + Alt + Space:
- якщо панель закрита, відкриває її;
- якщо панель відкрита, переходить до наступної кнопки.

Ctrl + Alt + E:
- відкриває вікно налаштувань.

У самій панелі:
- Enter виконує вибрану кнопку;
- Esc закриває панель;
- стрілки переміщують вибір по кнопках;
- Tab переходить до наступної кнопки.


2.3. Налаштування

У вікні налаштувань можна:
- вибрати сторінку кнопок;
- змінити назву сторінки;
- додати нову сторінку;
- змінити назву кожної кнопки;
- вибрати тип дії кнопки;
- вказати значення дії;
- вибрати файл або папку через Провідник;
- додати картинку для кнопки;
- змінити позицію панелі;
- змінити розмір сітки;
- змінити тему;
- імпортувати сет;
- експортувати сет;
- зберегти налаштування;
- закрити налаштування;
- повністю вийти з програми.

Коли відкривається вікно налаштувань, сама панель автоматично ховається.


2.4. Теми

У програмі доступні теми:
- поточна темна;
- кавова;
- бежева;
- сіро-синя.

Тема застосовується до панелі та вікна налаштувань.
Вибрана тема зберігається у файлі data\buttons.txt.


2.5. Імпорт та експорт сетів

Сет - це txt-файл з налаштуваннями кнопок.
Його можна експортувати і передати іншому користувачу.
Інший користувач може імпортувати цей txt-файл у своїй програмі.

У сет входить:
- сторінки;
- кнопки;
- типи дій;
- посилання;
- шляхи до програм, файлів або папок;
- гарячі клавіші;
- текстові дії;
- шляхи до картинок;
- розмір сітки;
- позиція панелі;
- тема.

Кнопка "Експорт сету":
- відкриває вікно збереження файлу;
- зберігає поточний сет у txt-файл.

Кнопка "Імпорт сету":
- відкриває вікно вибору txt-файлу;
- завантажує вибраний сет;
- застосовує його до програми;
- зберігає його як поточний data\buttons.txt.


3. Файл даних data\buttons.txt
------------------------------

Основні дані програми зберігаються у файлі:

data\buttons.txt

Файл зберігається у форматі UTF-8 з BOM, щоб український текст працював коректно.

Приклад службових рядків:

SETTINGS|position|right_center
SETTINGS|grid|3
SETTINGS|theme|dark

Приклад сторінки:

PAGE|Головна
YouTube|website|https://youtube.com|
Notepad|program|notepad.exe|
Documents|folder|%USERPROFILE%\Documents|

Формат кнопки:

Назва|тип|значення|шлях_до_картинки

Типи кнопок:
- website - відкриває сайт;
- program - запускає програму або файл;
- folder - відкриває папку;
- hotkey - виконує комбінацію клавіш;
- text - вводить текст;
- next_page - переходить на іншу сторінку;
- none - вимикає кнопку.


4. Структура файлів проєкту
---------------------------

4.1. kursova.cpp

Головний файл програми.
Відповідає за:
- точку входу wWinMain;
- запуск GDI+;
- ініціалізацію COM;
- реєстрацію головного вікна панелі;
- реєстрацію вікна налаштувань;
- створення головного прихованого вікна програми;
- головний цикл повідомлень Windows;
- основну логіку Win32-вікон панелі та редактора.

Також у цьому файлі залишена логіка:
- малювання панелі;
- анімації відкриття/закриття;
- обробки кліків по кнопках;
- обробки клавіш на панелі;
- створення редактора налаштувань;
- обробки команд у редакторі.


4.2. App.h

Заголовковий файл зі спільними типами, константами та глобальними змінними.
Відповідає за опис того, з чим працює вся програма.

Містить:
- ідентифікатори контролів;
- константи сітки;
- enum ActionType;
- enum PanelPosition;
- enum AppTheme;
- структуру ThemeColors;
- структуру ButtonAction;
- структуру Page;
- структуру ButtonEditorControls;
- оголошення глобальних змінних;
- оголошення допоміжних функцій.


4.3. App.cpp

Реалізація спільної логіки програми.
Відповідає за:
- глобальний стан програми;
- розрахунок кількості кнопок;
- розрахунок розміру панелі;
- обмеження розміру сітки від 3 до 6;
- кольори тем;
- оновлення пензлів для вікна налаштувань;
- створення стандартних кнопок при першому запуску;
- нормалізацію сторінок до максимальної кількості кнопок;

4.5. Config.cpp

Відповідає за:

5. Потік роботи програми
------------------------

1. Запускається wWinMain у kursova.cpp.
2. Ініціалізується COM.
3. Ініціалізується GDI+.
4. Викликається LoadButtons.
5. Програма читає data\buttons.txt.
6. Якщо файл не знайдено або він некоректний, створюються стандартні кнопки.
7. Реєструється клас вікна панелі.
8. Реєструється клас вікна налаштувань.
9. Створюється головне приховане вікно.
10. Реєструються глобальні hotkey.
11. Додається іконка в трей.
12. Запускається цикл повідомлень Windows.


6. Як працює кнопка
-------------------

Користувач натискає кнопку на панелі.
Програма визначає номер кнопки через HitTestButton.
Потім викликається ExecuteButton.

ExecuteButton дивиться на тип кнопки:
- website/program/folder -> ShellExecuteW;
- hotkey -> SendInput для комбінації клавіш;
- text -> SendInput з Unicode-символами;
- next_page -> SwitchToPage;
- none -> нічого не робить.

Після виконання кнопки панель ховається.


7. Як працює імпорт сету
------------------------

1. Користувач натискає "Імпорт сету".
2. Відкривається діалог вибору txt-файлу.
3. Викликається LoadButtonsFromFile.
4. Дані з файлу завантажуються у пам'ять.
5. Оновлюється редактор.
6. Оновлюється панель.
7. Сет зберігається як поточний data\buttons.txt.


8. Як працює експорт сету
-------------------------

1. Користувач натискає "Експорт сету".
2. Поточні дані з редактора збираються через CollectEditorPage.
3. Відкривається діалог збереження txt-файлу.
4. Викликається SaveButtonsToFile.
5. У вибраний файл записується повний сет.


9. Як додати новий тип кнопки в майбутньому
------------------------------------------

Щоб додати новий тип дії:

1. Додати новий варіант в enum ActionType у App.h.
2. Оновити ParseActionType у App.cpp.
3. Оновити ActionTypeToString у App.cpp.
4. Додати новий пункт у FillActionCombo у kursova.cpp.
5. Оновити ShowHelpForAction у kursova.cpp.
6. Додати виконання нового типу в ExecuteButton у Actions.cpp.


10. Як додати нову тему
-----------------------

Щоб додати тему:

1. Додати новий варіант в enum AppTheme у App.h.
2. Додати кольори в CurrentThemeColors у App.cpp.
3. Додати парсинг у ParseTheme.
4. Додати текстове значення у ThemeToString.
5. Додати назву для UI у ThemeToLabel.
6. Додати пункт у FillThemeCombo у kursova.cpp.


11. Як додати нове положення панелі
----------------------------------

Щоб додати нову позицію:

1. Додати варіант у enum PanelPosition у App.h.
2. Додати парсинг у ParsePanelPosition у App.cpp.
3. Додати запис у PanelPositionToString у App.cpp.
4. Додати назву в PanelPositionToLabel у App.cpp.
5. Додати пункт у FillPanelPositionCombo у kursova.cpp.
6. Додати розрахунок координат у GetPanelTargetRect у kursova.cpp.


12. Залежності Windows API
--------------------------

Програма використовує Win32 API:
- CreateWindowW / CreateWindowExW - створення вікон і контролів;
- RegisterHotKey - глобальні гарячі клавіші;
- ShellExecuteW - запуск сайтів, програм, файлів, папок;
- SendInput - емуляція клавіатури;
- Shell_NotifyIconW - іконка в треї;
- GetOpenFileNameW - вибір файлу;
- GetSaveFileNameW - збереження файлу;
- SHBrowseForFolderW - вибір папки;
- GDI/GDI+ - малювання панелі та картинок;
- DwmSetWindowAttribute - округлення вікна.


13. Що важливо пам'ятати
------------------------

- Перед збіркою нової версії треба повністю закрити стару програму.
- Якщо kursova.exe ще запущений, Visual Studio не зможе перезаписати exe.
- Закривати програму краще через кнопку "Вийти" у налаштуваннях або через трей.
- data\buttons.txt є основним поточним конфігом.
- Експортований txt-файл можна передавати іншому користувачу як сет.
- Шляхи до картинок у сеті можуть не працювати на іншому комп'ютері, якщо там немає таких самих файлів у таких самих шляхах.
